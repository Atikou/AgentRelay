# AgentRelay 项目验收清单（人类专用）

> **唯一验收依据**：你验收大模块（计划模式、Agent 模式、M1–M8 等）时**只认本文件**，不认 `agent-todolist.md` 的 `[x]`、不认 Agent 口头「已完成」、不认单测通过即模块结案。  
> **勾选权**：**仅人类**可在本文件打勾；Agent / CI / 脚本**禁止**修改本文件勾选状态。  
> **完成定义**：某模块下所有相关条目均为 `[x]`，且模块末尾「模块签收」由你签字/打勾，该模块才可验收。  
> **实现进度**：开发过程用 `agent-todolist.md`、`docs/*-TodoList.md` 跟踪；与本清单**解耦**——代码落地 ≠ 验收通过。

**来源合并**：`agent_acceptance_checklist.md`（对话 + 工程双维度）、`agent-todolist.md`（全量能力）、`Agent_TS_实现指南_修订版.md` 附录 C、`docs/*-TodoList.md`、计划架构断点分析。

**建议验收顺序**：§0 总闭环 → §1 对话能力 → §2 Agent 主模式 → §3 计划模式 → 其余模块按依赖排列。

## §0 总闭环（全项目一票否决项）

以下任一项未勾选，**整个项目不可总验收**：

- [ ] **可执行闭环**：`User → Plan → Compile → ToolGraph → Execute → Side-effect → Verify → Trace` 在测试台可端到端演示（非 no-op）
- [ ] **计划三步语义统一**：Chat 只读计划 / 计划工作流 / Draft Planner 边界清晰，用户不会误把 Markdown 当执行体
- [ ] **每个可执行 step 必有** `tool` + `toolInput`（或等价 `toolName` + `args`），Executor **禁止**对无工具步骤静默跳过
- [ ] **副作用可验证**：至少一条验收用例证明写文件或 shell 结果真实落盘
- [ ] **失败可复盘**：同一 run 可从 trace / timeline / run report 还原决策链
- [ ] **高风险默认拦截**：删除、推送、安装依赖等须确认门，非 dry-run 不可系统 auto-approve
