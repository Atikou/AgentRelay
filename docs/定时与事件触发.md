# 定时与事件触发（M8）



M8 引入 **`Scheduler`**：一次性、周期性、cron 与事件触发；触发后写入通知队列，并通过 `Orchestrator` 创建 `scheduled` Run。无人值守白名单命中时可进入 AgentLoop，否则保持待确认的 pending Run，不绕过权限直接执行工具。



## 触发器类型



| kind | 说明 |

| --- | --- |

| `once` | ISO 时间 `at`；`missPolicy`: `skip` / `run_once` |

| `interval` | `intervalMs` 周期；最小 1000ms |

| `cron` | croner 表达式；`cronMissPolicy`: `skip` / `run_once`（启动时补一次） |

| `event` | 见下表 |



| eventType | 过滤 |

| --- | --- |

| `background_completed` | 可选 `eventFilter.status` |

| `file_changed` | `watchPath`、`pattern`（`*.ts`）、`debounceMs`（最小 100ms） |

| `git_changed` | `dirtyOnly`、`branch`；轮询 `git status` |



## 配置（`config/*.json` → `scheduler`）



```json

{

  "scheduler": {

    "unattendedGoalPatterns": ["周期巡检"],

    "gitPollIntervalMs": 5000,

    "cronMissPolicy": "skip",

    "dailySummaryCron": "0 9 * * *",

    "dailySummaryGoal": "生成昨日变更摘要（只读）"

  }

}

```



- **无人值守白名单**：`goal` 匹配 `unattendedGoalPatterns` 时通知 `requiresConfirmation: false`。

- **daily_summary**：启动时若无 `__daily_summary__` 触发器则自动注册 cron。

## 与 Run / 通知的关联

`Scheduler.setFireHandler()` 由 `AppContext` 接到 `Orchestrator`：

- 普通触发：调用 `createScheduledRun()`，创建 `status=pending` 的 `scheduled` Run，并把 `runId` 写入调度通知。
- 无人值守触发：调用 `executeUnattendedTrigger()`，创建 `status=running` 的 `scheduled` Run 并异步执行 AgentLoop。
- 调度通知包含 `priority`、`dedupeKey`、`mergeKey`、`triggerId`、`requiresConfirmation`、`unattended`，便于后续待办队列按优先级展示和合并。



## HTTP API



| 方法 | 路径 | 说明 |

| --- | --- | --- |

| GET | `/api/scheduler/triggers` | 列出触发器 |

| POST | `/api/scheduler/triggers` | 注册 |

| POST | `/api/scheduler/triggers/:id/pause|resume|cancel` | 控制状态 |
