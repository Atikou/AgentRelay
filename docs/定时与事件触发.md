# 定时与事件触发（M8）

M8 引入 **`Scheduler`**：一次性、周期性、cron 与事件触发；触发后**只向通知队列写入待办**，不绕过权限直接执行工具。

## 触发器类型

| kind | 说明 |
| --- | --- |
| `once` | ISO 时间 `at`；`missPolicy`: `skip` / `run_once` |
| `interval` | `intervalMs` 周期 |
| `cron` | croner 表达式；`cronMissPolicy`: `skip` / `run_once`（启动时补一次） |
| `event` | 见下表 |

| eventType | 过滤 |
| --- | --- |
| `background_completed` | 可选 `eventFilter.status` |
| `file_changed` | `watchPath`、`pattern`（`*.ts`）、`debounceMs` |
| `git_changed` | `dirtyOnly`、`branch`；轮询 `git status` |

## 配置（`config/*.json` → `scheduler`）

```json
{
  "scheduler": {
    "unattendedGoalPatterns": ["周期巡检"],
    "gitPollIntervalMs": 5000,
    "cronMissPolicy": "skip",
    "dailySummaryCron": "0 9 * * *",
    "dailySummaryGoal": "生成昨日变更摘要（只读）"
  }
}
```

- **无人值守白名单**：`goal` 匹配 `unattendedGoalPatterns` 时通知 `requiresConfirmation: false`。
- **daily_summary**：启动时若无 `__daily_summary__` 触发器则自动注册 cron。

## HTTP API

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/scheduler/triggers` | 列出触发器 |
| POST | `/api/scheduler/triggers` | 注册 |
| POST | `/api/scheduler/triggers/:id/pause|resume|cancel` | 控制状态 |

## 待办队列 UI

测试台 **「定时与触发」** 面板下半部展示 `source=scheduler` 的待处理通知。

## 自检

```bash
npm run test:scheduler    # 11 项
npm run test:integration  # 含后台通知注入
npm test                  # 全量 96 项
```
