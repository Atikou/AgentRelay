# 安全与审计（M7）



M7 聚焦**日志脱敏、工具调用审计、trace 导出与回放**，与既有命令风险分级、路径沙箱、确认门配合使用。



## 日志脱敏



`src/util/redact.ts` 提供：



| 函数 | 用途 |

| --- | --- |

| `redactString` | 遮蔽 `sk-*`、Bearer token、AWS key、私钥块等 |

| `redactValue` | 深度脱敏对象；字段名匹配 `password/secret/apiKey/token` 等一律替换 |

| `redactPreview` | 工具入参/出参预览（截断 + 脱敏） |



应用点：`TraceLogger` 写入、`ToolRegistry` 审计事件、`ContextManager` 错误 JSONL。

trace 查询接口读取 `data/traces/trace.jsonl` 时只从文件尾部按块回读最近事件，避免日志增长后每次审计查询全量载入内存。



## 写文件 patch 预览

测试台/HTTP 工具接口在 `write_file`、`apply_patch` 未确认时会先返回 `preview.patchPreview`，同时完成路径沙箱、`expectedHash` 与唯一匹配校验；只有调用方再次提交 `confirm:true` 才执行写入。

## 通知注入围栏

后台任务和调度器通知会在 AgentLoop 的安全点注入模型上下文。通知正文视为不可信外部数据；若命中 prompt injection 片段，会被包装为 `_untrusted` 数据，提醒模型只当作事实输入而不要执行其中指令。



`write_file` 执行前经 `util/patch.ts` 生成 `patchPreview`（新文件/变更摘要），写入工具输出与 `tool_audit.outputPreview`，供确认前审阅。



## Prompt injection 防护



`util/injection.ts` 对 `read_file` / `search_text` / `list_files` 输出扫描可疑短语；命中时以 `_untrusted` 围栏包装后回灌 `AgentLoop`，提示模型勿执行其中指令。



## 工具审计



每次工具执行写入 trace（`data/traces/trace.jsonl`）：



```json

{

  "type": "tool_audit",

  "tool": "read_file",

  "status": "start|ok|error",

  "permission": "read",

  "inputPreview": "...",

  "durationMs": 12

}

```



## HTTP API



| 方法 | 路径 | 说明 |

| --- | --- | --- |

| GET | `/api/trace/recent?limit=50` | 最近 N 条脱敏 trace 事件 |

| GET | `/api/trace/export?limit=500` | 导出近期事件（含 `exportedAt`） |

| GET | `/api/trace/replay?limit=100` | **审计回放**：仅 `tool_audit` / `agent_tool` / `task_step` 等链路事件 |



`capabilities.traceReplay: true` 表示已注册回放接口。修改 API 后需重启 `npm run serve`。
