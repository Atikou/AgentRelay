# 自审核记录

每次结束任务时，对照「当前框架 / 预定效果 / 是否缺失功能模块」做一次自审核，并把结论写入本文件。

## 写入规范

- **位置**：新记录一律写在文件头（紧跟本规范说明之后、旧记录之前），保持**最新在上**。
- **标题格式**：`### 时间_目标_模型_本次任务概括`
  - 时间：`YYYY-MM-DD HH:mm`
  - 目标：本轮任务要达成的目标
  - 模型：执行本轮任务的模型名
  - 本次任务概括：一句话概括做了什么
- **正文建议包含**：
  - 本次改动清单（文件 / 模块）
  - 是否符合当前框架与分层约定
  - 是否达到预定效果
  - 是否缺失功能模块 / 已知缺口 / 后续待办
  - 自检结果（typecheck / 测试 / 文档站等）

> 第一次预览项目时也应浏览本文件，了解项目演进与已知缺口。
