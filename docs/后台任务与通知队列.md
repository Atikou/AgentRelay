# 后台任务与通知队列（M4）

AgentRelay 支持将长时间 Shell 命令放到**后台**运行，并在任务完成时通过**通知队列**异步告知主 Agent。主 Agent 仅在**安全点**消费通知，避免在写文件或执行工具中途被打断。

## 能力概览

| 能力 | 说明 |
| --- | --- |
| 后台启动 | `spawn` + `shell`，不阻塞 HTTP / 主循环 |
| 输出记录 | 累积 stdout / stderr（单流上限 512KB，保留尾部） |
| 状态查询 | `running` / `completed` / `failed` / `cancelled` |
| 取消任务 | Windows 使用 `taskkill /T /F` 终止进程树 |
| 通知队列 | 任务结束时写入；JSONL 持久化到 `data/notifications/` |
| 安全点消费 | AgentLoop 在「循环开始」与「每次工具完成后」drain 通知 |

## HTTP 接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/api/background/start` | body: `{ "command": "npm run typecheck", "cwd?": "agent-relay" }` |
| `GET` | `/api/background` | 列出全部后台任务 |
| `GET` | `/api/background/:id` | 查询单个任务（含输出） |
| `POST` | `/api/background/:id/cancel` | 取消运行中的任务 |
| `GET` | `/api/notifications?pending=1` | 列出通知（`pending=1` 仅未消费） |
| `POST` | `/api/notifications/consume` | 手动在安全点消费全部待处理通知 |

`POST /api/agent` 的智能体循环已接入通知队列：返回体中的 `notifications` 字段为本轮在安全点已消费的通知列表。

## 安全点定义

与实现指南一致，当前在以下位置消费通知：

1. **AgentLoop 每次 `run()` 开始前**（承接上一轮遗留通知）
2. **每个工具步骤执行完成后**（下一轮模型调用前）
3. **测试台「通知队列 → 手动消费」**（调试 / 非 Agent 场景）

不在模型推理中途、工具执行中途注入通知。

## 测试台

启动 `npm run serve` 后，左侧：

- **后台任务**：输入命令后台启动、刷新列表、取消运行中任务
- **通知队列**：查看待处理通知、手动消费

## 代码位置

```text
agent-relay/src/background/
├─ BackgroundTaskManager.ts   # spawn、输出、取消、完成时 enqueue

├─ types.ts
└─ index.ts
```

自检：`npm run test:background`（5 项）。

## 与 shell_run 的区别

| | `shell_run` 工具 | 后台任务 API |
| --- | --- | --- |
| 阻塞 | 同步等待命令结束（最长 60s） | 立即返回 task id |
| 适用 | 短命令、Agent 需要即时结果 | `npm run dev`、`build`、长时间测试 |
| 通知 | 无 | 完成时写入通知队列 |
