# 模型路由与单任务多模型协作

实现依据：`Agent_Model_Router_Collaboration_Spec.md`（规则路由 + 手动等级配置 + 有限协作）。

**升级路线与待办**：见 [模型路由升级 TodoList](模型路由升级TodoList.md)（**V2 FallbackManager 已落地**；V3 启发式部分接入；下一步 V4 AnswerEvaluator）。

## 双轨路由边界（必读）

仓库内并存两套路由器，**职责不同、互不替代**。改路由相关代码前请先确认改的是哪一层。

| 维度 | `model/ModelRouter`（客户端层） | `SmartModelRouter`（任务层） |
|------|-------------------------------|------------------------------|
| **源码** | `src/model/ModelRouter.ts` | `src/model-router/smart-model-router.ts` + `DecisionEngine` + `ModelRegistry` |
| **选型依据** | `config.routing.strategy`（local-first / cloud-first / privacy-first 等）+ 粗粒度 `taskType` 提示 | `RuleRouter` 关键词/质量模式 + `routerProfile` 等级 + 协作策略 |
| **输出** | 直接 `ModelResponse`（一次 chat） | `RouterDecision`（含 `executionStrategy`、候选模型 id）→ 交 `ModelOrchestrator` 执行 |
| **失败处理** | **连通性 fallback**：按候选列表依次尝试下一客户端 | **任务级升级**：`FallbackManager`（空输出/审查拒绝等 → 强模型，最多 2 次） |
| **协作** | 不支持 draft+review | 支持 `local_draft_remote_review`、`rule_only` 等 |
| **高风险** | 无审查模型时仍可能换候选客户端 | 无 review 模型时 **不静默降级**（`NO_REVIEW_MODEL_AVAILABLE`） |
| **日志** | `MetricsRegistry` + `TraceLogger` | `model_route_logs` / `model_call_logs` / `fallback_logs` 等 |
| **隐私** | `sensitive` / `privacy-first` 仅本地；remote 调用前脱敏 | 继承 `localOnly`；fallback 不升级到远程 |

### 何时走哪条轨

| HTTP / 模块 | 默认（未指定 `clientName`） | 显式 `clientName` / `forceClient` |
|-------------|----------------------------|-------------------------------------|
| `POST /api/chat` | **Smart** + `ModelOrchestrator`（含协作与 fallback） | 绕过 Smart，直连指定客户端（`ModelRouter` 语义） |
| `POST /api/agent` | **Smart 单模型**（`createAgentChatFn`，每轮 ReAct 不走协作流水线） | `ModelRouter` 强制该客户端 |
| `Planner` / `/api/plans/*` | **Smart 单模型**（`createPlannerChatFn`） | `ModelRouter` 强制该客户端 |
| `SubAgentCoordinator` | **Smart 单模型**（`defaultAgentChat`） | `makeChatFn(forceClient)` → `ModelRouter` |
| 无人值守 / Scheduler 触发 Agent | 同 Agent 默认 Smart 路径 | 同 Agent forceClient 路径 |

> **记忆口诀**：`ModelRouter` = **连哪台机、挂了换谁**；`SmartModelRouter` = **这任务该什么等级、什么策略、要不要协作**。

### 语义差异示例

- **连通性 fallback**（`ModelRouter`）：远程 API 超时 → 尝试列表中下一个客户端（可能是本地）。
- **任务级升级**（`FallbackManager`）：架构协作审查拒绝 → 升级为 `strong_model_direct`，并写 `fallback_logs`；draft/review 中间结果不进 `messages`。
- 二者可串联：Agent 默认先用 Smart 选出 `selectedModelId`，实际 HTTP 调用仍经 `ModelChatFn`；指定 `clientName` 时则完全跳过 Smart 决策。

## 流程

```text
用户输入 → ContextRestorer → SmartModelRouter.route()
  → RuleRouter（任务类型/风险/等级）
  → DecisionEngine + ModelRegistry（选模型与策略）
  → ModelOrchestrator（single_model / rule_only / strong_model_direct / local_draft_remote_review）
    → FallbackManager（失败/空输出/审查拒绝时升级模型，最多 2 次）
  → 仅 saveAssistantMessage(finalAnswer)
```

## 模块

| 目录 | 职责 |
|------|------|
| `src/model/ModelRouter.ts` | **客户端层**：`config.routing.strategy` 排序 + **连通性 fallback** + 显式 `forceClient`；**不负责**任务规则与协作策略 |
| `src/model-router/`（Smart 栈） | **任务层**：`RuleRouter`、`DecisionEngine`、`ModelRegistry`、`FallbackManager`、`SmartModelRouter`、SQLite 路由/调用/fallback 日志 |

| `src/model-router/create-planner-chat.ts` | 计划模式单模型路由（`createPlannerChatFn`） |

## Level 0 规则直答（`rule_only`）

短问候（如「你好」「谢谢」）经 `RuleRouter` 判定为 `requiredLevel: 0`、`executionStrategy: rule_only`：

- `DecisionEngine` 不选模型，`candidates` 为空
- `ModelOrchestrator` 走 `runRuleOnlyPipeline`，返回固定文案
- 不写入 `model_call_logs`；Agent 单轮路径（`createSmartSingleModelChatFn`）同样支持

`qualityMode=fast` 或显式需模型时仍强制 `single_model`；含「闲聊」等长句的 casual 任务仍走 Level 1 轻量模型。

## V3/V4 评估器

- `src/model-router/router-model-evaluator.ts`：`RouterModelEvaluator` 在 `unknown` 等不确定任务上可覆盖规则（`source=evaluator`）；**高风险任务（`risk=high` / `high_risk_action`）不覆盖**。
- `src/model-router/answer-evaluator.ts`：`AnswerEvaluator` 规则版已接入 `ModelOrchestrator` 与 `FallbackManager`；识别空输出与复杂任务过短回答，触发 `FallbackManager` 升级并在 `fallback_logs.reason` 记录 `V4 评估` 原因。
- `DecisionEngine` 与 `ModelOrchestrator` 中评估结果只写路由/fallback 日志，不污染最终 `messages`。

## V6 RuntimeStats（只读建议）

`RuntimeStatsCollector`（`src/model-router/runtime-stats.ts`）聚合：

| 数据源 | 指标 |
|--------|------|
| `model_call_logs` | 按 `modelId` 的调用数、错误率、平均耗时、token |
| `fallback_logs` | 各模型作为 fallback 源/目标的次数 |
| `model_route_logs` | 按 `taskType` 的 fallback 比例、`source=evaluator` 次数、主流 `executionStrategy` |
| `MetricsRegistry` | 进程内客户端连通性失败率（与 SQLite 路由统计对照） |

**HTTP**：`GET /api/routing/stats?limit=200` 返回 `summary`、`models`、`taskTypes`、`processMetrics`、`suggestions`。

- **只建议、不改配置**：`suggestions` 为规则版提示（如模型错误率偏高、某 taskType fallback 过多），不会自动修改 `config/*.json` 或 `routerProfile`。
- 测试台「模型路由日志」面板提供 **运行统计** 按钮，展示建议与模型表。

`capabilities.runtimeStatsV6: true` 声明已落地。

## V7 EvalSetRunner（离线评测）

`EvalSetRunner`（`src/model-router/eval-set-runner.ts`）对固定评测集离线跑 **RuleRouter**（`scope=rule`）或 **RuleRouter + DecisionEngine**（`scope=smart`），**不调用模型、不写 `model_route_logs`**。

| HTTP | 说明 |
|------|------|
| `POST /api/routing/eval/run` | 运行评测；默认可用 `DEFAULT_ROUTING_EVAL_SET`（12 条） |
| `GET /api/routing/eval/runs` | 历史运行列表 |
| `GET /api/routing/eval/runs?runId=` | 单次运行详情与逐条 `results` |

结果持久化到 `memory.db` 的 `model_eval_runs` / `model_eval_results`（schema v8）。测试台「模型路由日志」面板提供 **离线评测** 按钮。

`capabilities.evalSetRunnerV7: true` 声明已落地。全量自动评测与配置自动调优仍不在当前阶段。

## 配置

每个 `config/*.json` 的 `models.clients[]` 可选手动能力块 `routerProfile`（省略时按 `location`/模型名推断 Level 与 draft/review 角色）：

```json
{
  "name": "local-qwen35",
  "location": "local",
  "routerProfile": {
    "defaultLevel": 1,
    "relativeCost": "free",
    "canDraft": true,
    "canReview": false,
    "canFinal": true,
    "allowedRoles": ["primary", "draft"]
  }
}
```

## HTTP：`GET /api/routing/logs`

| 参数 | 说明 |
|------|------|
| `limit` | 最近路由条数（默认 20，最大 100） |
| `sessionId` | 按会话过滤 |
| `routeLogId` | 指定单条时返回关联 `calls` / `collaborations` / `fallbacks` |

测试台侧栏的「模型路由日志」面板会读取该接口：默认展示最近路由决策，可按 `sessionId` 过滤；点击单条记录的「详情」可查看关联的 `calls`、`collaborations` 与 `fallbacks`；**运行统计** 按钮调用 `GET /api/routing/stats` 查看 V6 聚合建议。

## HTTP：`GET /api/routing/stats`

| 参数 | 说明 |
|------|------|
| `limit` | 聚合最近路由条数（默认 200，最大 1000） |

响应含 `models`（按模型）、`taskTypes`（按任务类型）、`processMetrics`（进程内指标）、`suggestions`（只读调优建议，**不修改配置**）。

## HTTP：`POST /api/chat`

自动路由（未指定 `clientName`）时启用 `SmartModelRouter` + `ModelOrchestrator`：

| 字段 | 说明 |
|------|------|
| `qualityMode` | `fast` / `balanced` / `deep` |
| `allowCollaboration` | 默认 true；`fast` 时禁用协作 |
| `forceSingleModel` | 强制单模型 |
| `hasAttachments` / `attachmentTypes` | 图片 → 需 `supportsVision` |
| `sensitive` | 仅本地模型；智能路由与 fallback 均继承 `localOnly` |
| `taskType` | （兼容）映射旧枚举到规则任务类型 |

响应额外字段：`routerDecision`、`executionStrategy`、`collaborationRunId`、**`fallbackCount`** / **`fallbackLogIds`**（V2，仅发生模型升级时返回）。

> `sensitive=true` 会强制仅本地；除此之外，旧 `ModelRouter` 在真正调用 remote client 前也会对消息内容做敏感信息脱敏，防止 API key/token/私钥片段直接发往云端。

## 本地优先隐私模式

- 单次请求：`POST /api/chat` 传 `sensitive:true` 时，`ModelRouter`、`SmartModelRouter` 和 `FallbackManager` 都只允许选择本地模型。
- 全局 profile：`config/local-only.json` 使用 `routing.strategy=privacy-first`，适合需要默认禁止远程模型调用的环境。
- 失败升级：敏感任务的 fallback 不会从本地模型升级到远程 API 模型；若没有可用本地候选，会返回“没有满足当前策略的候选模型”一类错误。

`GET /api/config.capabilities.localFirstPrivacyMode` 为 `true` 时，表示以上隐私约束已在当前进程中启用。

## 协作与执行策略

- **rule_only**：Level 0 规则直答，不调用模型（见上文「Level 0」）。
- **single_model**：一次调用，结果即最终回复。
- **strong_model_direct**：强模型（Level 3）直答；通常由 V2 fallback 升级触发。
- **local_draft_remote_review**：草稿模型 → 审查模型 JSON（`approve`/`revise`/`reject`）→ 仅保存 `finalAnswer`；失败时由 `FallbackManager` 记录并升级。

中间模型输出写入 `model_call_logs` / `model_collaboration_runs` / **`fallback_logs`**，**不**写入 `messages`。

## V2 FallbackManager

触发条件（节选）：`model_error`、`empty_output`、`json_parse_failed`、`review_rejected`、`answer_too_short`。

升级路径（节选）：

- L1 `single_model` → L2 → L3 `strong_model_direct`
- `local_draft_remote_review` 草稿失败 / 审查拒绝 → `strong_model_direct`

硬限制：**单次请求最多 2 次 fallback**（`MAX_FALLBACKS_PER_REQUEST`）。

## 数据库（schema v7）

- `model_route_logs` — 路由决策
- `model_call_logs` — 每次模型调用（含 role: draft/review/primary）
- `model_collaboration_runs` — 协作 verdict/issues
- **`fallback_logs`** — fallback 链（`route_log_id` 关联）
