# 模型路由与单任务多模型协作

实现依据：`Agent_Model_Router_Collaboration_Spec.md`（规则路由 + 手动等级配置 + 有限协作）。

**升级路线与待办**：见 [模型路由升级 TodoList](模型路由升级TodoList.md)（当前 V1 基本完成，下一步 V2 FallbackManager）。

## 流程

```text
用户输入 → ContextRestorer → SmartModelRouter.route()
  → RuleRouter（任务类型/风险/等级）
  → DecisionEngine + ModelRegistry（选模型与策略）
  → ModelOrchestrator（single_model 或 local_draft_remote_review）
  → 仅 saveAssistantMessage(finalAnswer)
```

## 模块

| 目录 | 职责 |
|------|------|
| `src/model-router/` | 类型、`buildModelProfiles`、`RuleRouter`、`ModelRegistry`、`DecisionEngine`、`SmartModelRouter`、SQLite 路由/调用日志 |
| `src/model-orchestrator/` | `ModelOrchestrator`、`SingleModelPipeline`、`DraftReviewPipeline` |
| `src/model/ModelRouter.ts` | 旧客户端级 fallback（Agent 循环、指定 `clientName` 时仍使用） |

## 配置

每个 `config/*.json` 的 `models.clients[]` 可选手动能力块 `routerProfile`（省略时按 `location`/模型名推断 Level 与 draft/review 角色）：

```json
{
  "name": "local-qwen35",
  "location": "local",
  "routerProfile": {
    "defaultLevel": 1,
    "relativeCost": "free",
    "canDraft": true,
    "canReview": false,
    "canFinal": true,
    "allowedRoles": ["primary", "draft"]
  }
}
```

## HTTP：`POST /api/chat`

自动路由（未指定 `clientName`）时启用 `SmartModelRouter`：

| 字段 | 说明 |
|------|------|
| `qualityMode` | `fast` / `balanced` / `deep` |
| `allowCollaboration` | 默认 true；`fast` 时禁用协作 |
| `forceSingleModel` | 强制单模型 |
| `hasAttachments` / `attachmentTypes` | 图片 → 需 `supportsVision` |
| `sensitive` | 仅本地模型 |
| `taskType` | （兼容）映射旧枚举到规则任务类型 |

响应额外字段：`routerDecision`、`executionStrategy`、`collaborationRunId`。

## 协作策略

- **single_model**：一次调用，结果即最终回复。
- **local_draft_remote_review**：草稿模型 → 审查模型 JSON（`approve`/`revise`/`reject`）→ 仅保存 `finalAnswer`；draft/review 写入 `model_call_logs` / `model_collaboration_runs`，**不**写入 `messages`。

## 数据库（schema v3）

- `model_route_logs` — 路由决策
- `model_call_logs` — 每次模型调用（含 role: draft/review/primary）
- `model_collaboration_runs` — 协作 verdict/issues
