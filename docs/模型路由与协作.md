# 模型路由与单任务多模型协作

实现依据：`Agent_Model_Router_Collaboration_Spec.md`（规则路由 + 手动等级配置 + 有限协作）。

**升级路线与待办**：见 [模型路由升级 TodoList](模型路由升级TodoList.md)（**V2 FallbackManager 已落地**；下一步 V3 RouterModelEvaluator）。

## 流程

```text
用户输入 → ContextRestorer → SmartModelRouter.route()
  → RuleRouter（任务类型/风险/等级）
  → DecisionEngine + ModelRegistry（选模型与策略）
  → ModelOrchestrator（single_model / strong_model_direct / local_draft_remote_review）
    → FallbackManager（失败/空输出/审查拒绝时升级模型，最多 2 次）
  → 仅 saveAssistantMessage(finalAnswer)
```

| 能力 | 阶段 | 说明 |

## 配置

每个 `config/*.json` 的 `models.clients[]` 可选手动能力块 `routerProfile`（省略时按 `location`/模型名推断 Level 与 draft/review 角色）：

```json
{
  "name": "local-qwen35",
  "location": "local",
  "routerProfile": {
    "defaultLevel": 1,
    "relativeCost": "free",
    "canDraft": true,
    "canReview": false,
    "canFinal": true,
    "allowedRoles": ["primary", "draft"]
  }
}
```

## HTTP：`GET /api/routing/logs`

| 参数 | 说明 |
|------|------|
| `limit` | 最近路由条数（默认 20，最大 100） |
| `sessionId` | 按会话过滤 |
| `routeLogId` | 指定单条时返回关联 `calls` / `collaborations` / `fallbacks` |

## HTTP：`POST /api/chat`

自动路由（未指定 `clientName`）时启用 `SmartModelRouter` + `ModelOrchestrator`：

| 字段 | 说明 |
|------|------|
| `qualityMode` | `fast` / `balanced` / `deep` |
| `allowCollaboration` | 默认 true；`fast` 时禁用协作 |
| `forceSingleModel` | 强制单模型 |
| `hasAttachments` / `attachmentTypes` | 图片 → 需 `supportsVision` |
| `sensitive` | 仅本地模型；智能路由与 fallback 均继承 `localOnly` |
| `taskType` | （兼容）映射旧枚举到规则任务类型 |

响应额外字段：`routerDecision`、`executionStrategy`、`collaborationRunId`、**`fallbackCount`** / **`fallbackLogIds`**（V2，仅发生模型升级时返回）。

> `sensitive=true` 会强制仅本地；除此之外，旧 `ModelRouter` 在真正调用 remote client 前也会对消息内容做敏感信息脱敏，防止 API key/token/私钥片段直接发往云端。

## 本地优先隐私模式

- 单次请求：`POST /api/chat` 传 `sensitive:true` 时，`ModelRouter`、`SmartModelRouter` 和 `FallbackManager` 都只允许选择本地模型。
- 全局 profile：`config/local-only.json` 使用 `routing.strategy=privacy-first`，适合需要默认禁止远程模型调用的环境。
- 失败升级：敏感任务的 fallback 不会从本地模型升级到远程 API 模型；若没有可用本地候选，会返回“没有满足当前策略的候选模型”一类错误。

`GET /api/config.capabilities.localFirstPrivacyMode` 为 `true` 时，表示以上隐私约束已在当前进程中启用。

## 协作与执行策略

- **single_model**：一次调用，结果即最终回复。
- **strong_model_direct**：强模型（Level 3）直答；通常由 V2 fallback 升级触发。
- **local_draft_remote_review**：草稿模型 → 审查模型 JSON（`approve`/`revise`/`reject`）→ 仅保存 `finalAnswer`；失败时由 `FallbackManager` 记录并升级。

中间模型输出写入 `model_call_logs` / `model_collaboration_runs` / **`fallback_logs`**，**不**写入 `messages`。

## V2 FallbackManager

触发条件（节选）：`model_error`、`empty_output`、`json_parse_failed`、`review_rejected`、`answer_too_short`。

升级路径（节选）：

- L1 `single_model` → L2 → L3 `strong_model_direct`
- `local_draft_remote_review` 草稿失败 / 审查拒绝 → `strong_model_direct`

硬限制：**单次请求最多 2 次 fallback**（`MAX_FALLBACKS_PER_REQUEST`）。

## 数据库（schema v7）

- `model_route_logs` — 路由决策
- `model_call_logs` — 每次模型调用（含 role: draft/review/primary）
- `model_collaboration_runs` — 协作 verdict/issues
- **`fallback_logs`** — fallback 链（`route_log_id` 关联）
