# 数据生命周期与清理

> 对应实现：`agent-relay/src/lifecycle/` · 路线图：`docs/DataLifecycleRetention-TodoList.md`

## 概述

AgentRelay 默认**本地优先、追加写入**，长期使用后 trace、timeline、缓存会持续增长。数据生命周期模块提供：

- **用量统计**：`GET /api/storage/usage`
- **清理预览**（默认 dry-run）：`POST /api/storage/cleanup/preview`
- **安全清理执行**：`POST /api/storage/cleanup/apply`（需先 preview + `confirm: true`）
- **会话删除联动**：`DELETE /api/context/sessions/:id` 会清理关联 `.agent/runs/{runId}`、`data/runs/{runId}` 并写 `tombstones.jsonl`
- **隐私清除**：`POST /api/context/sessions/:id/purge`（需 `confirm: true`）不可恢复地清除 trace segments、tools.db、routing 表、通知 journal 等详细记录

## 策略文件

首次访问时自动写入：

```text
agent-relay/data/lifecycle/policy.json
```

默认保留天数（节选）：

| 类别 | TTL |
|------|-----|
| temp | 1 天 |
| file cache | 3 天 |
| report cache | 3 天 |
| 已消费通知 | 7 天 |

自动清理默认**关闭**（`cleanup.autoEnabled: false`）；apply 仅执行 **low** 风险动作。

**行级保留（`scope: "all"` 预览）**：

| 策略字段 | 行为 |
|----------|------|
| `softDeletedRows` | `delete_db_rows` 物理删除 `memories.is_active=0` 且超 TTL 的行（`scope: safe` 亦包含） |
| `routeDetails` | 删除超 TTL 的 `model_route_logs` / `model_call_logs` / `fallback_logs` 等 |
| `toolArgs` / `toolOutput` / `traceRaw*` | `rewrite_file` 裁剪 trace segment 内过期 `inputPreview`/`outputPreview`/`rawOutput` 等 verbose 字段 |
| `traceRawBytes` | 当 trace segments 总体积超配额时，预览生成按最旧优先的 `delete_file` 候选（`scope: all`） |

apply 成功后按 `sqlite.*` 策略执行 WAL checkpoint 与可选 VACUUM。

### Trace 分段布局

```text
data/traces/
  active/trace-current.jsonl    # 当前写入
  segments/YYYY/MM/trace-*.jsonl
  index.db                      # runId/sessionId → segment 索引
```

- 超过 `policy.trace.rotationMaxBytes` 或 `rotationMaxAgeHours` 自动轮转
- 可选 `policy.trace.compressOldSegments: true` 将封存段压缩为 `.jsonl.gz`（reader 自动解压）
- 进程退出时若有未轮转数据会封存到 segments
- 旧版 `trace.jsonl` 首次启动自动迁移到 segments 并建索引
- 手动轮转：`POST /api/trace/rotate`

- 删除单个 Run：`DELETE /api/runs/:runId`（联动 timeline + `data/runs/` + tombstone）
- 过期 timeline：`scope: "all"` 预览可看到 `events.jsonl` 清理候选（medium，保留 `summary.md`）

### 隐私清除（不可恢复）

```http
POST /api/context/sessions/{sessionId}/purge
Content-Type: application/json

{ "confirm": true }
```

流程：校验无运行中 Run → 轮转 active trace → 删除会话 → 重写 trace segments（过滤该 session/run 事件）→ 清理 `tools.db` / routing 表 / 通知 journal / timeline 落盘 → WAL checkpoint + 可选 VACUUM → 写入 `session_purge` tombstone。

## API 示例

### 查看占用

```http
GET /api/storage/usage
```

### 预览安全清理

```http
POST /api/storage/cleanup/preview
Content-Type: application/json

{ "scope": "safe" }
```

响应含 `cleanupRunId`、`summary.estimatedBytesToFree`、`actions[]`（含 `risk`、`canDelete`）。

### 执行清理

```http
POST /api/storage/cleanup/apply
Content-Type: application/json

{
  "cleanupRunId": "cleanup_20260613_abc123",
  "confirm": true
}
```

## 审计

- 每次 apply 动作追加到 `data/lifecycle/cleanup-runs.jsonl`
- 会话删除写入 `data/lifecycle/tombstones.jsonl`

## CLI

```bash
cd agent-relay
npm run storage:status
npm run storage:cleanup -- --dry-run
npm run storage:cleanup -- --apply --cleanup-run-id <id>
```

## 自动清理

在 `data/lifecycle/policy.json` 设置 `cleanup.autoEnabled: true` 后，服务端启动约 1 分钟首次执行，之后按 `autoIntervalHours`（默认 24h）自动 preview+apply 低风险项。
