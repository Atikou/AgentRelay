# 测试用例（网页验收）

除 `npm test`（含 `tests/http-e2e.test.ts` 真实 HTTP 监听自检）外，测试台按**里程碑 × 功能**提供独立用例页：每条含**测试目的**，可运行比对，并可**一键复制**给 AI 维护。

## 打开方式

```bash
cd agent-relay
npm run serve
```

打开独立测试用例页：

```text
http://localhost:18787/test-cases.html
```

也可以从测试台首页左侧 **测试用例** 或欢迎页按钮进入。测试用例不再渲染在主页右侧，而是单独打开 **AgentRelay 验收工作台**。

测试台首页右侧保持欢迎入口；历史会话位于左侧栏底部，点击会话会在右侧恢复该会话的历史对话，并自动滚动到最新一条消息。

## 页面布局

- **顶部导航**：返回测试台、说明文档、API 文档。
- **左侧功能导航**：按 `index.json` 顺序展示 M0→M9 功能页。
- **右侧工作区**：顶部展示功能页数量与用例总数；每个功能页含摘要、复制整页、运行整页、单条运行/填入/复制。
- **结果区**：展示输入、期望、实际输出三列比对；未填写期望时作为手动验证输出展示。
- **手动验证**：保留自定义 method/path 与复制为正式用例能力。

## 测试台 UI 维护约定

测试台的纯文本消息保留换行展示；工具面板、审计回放、运行报告、测试用例卡片等结构化 DOM 面板必须使用普通空白流。`addMessage()` 在插入 `Node` 内容时会给气泡添加 `structured-bubble`，对应 CSS 将 `white-space` 设为 `normal`，避免模板字符串缩进被浏览器渲染成大段空白，导致审计记录或列表卡片间距过大。

输入区的「高级选项」使用 `details > summary + .advanced-panel`：`advanced-panel` 负责纵向布局，`positionAdvancedPanel()` 以按钮左上角为锚点计算弹层位置，并按主区/视口边界夹紧，避免靠右时被裁切或靠左时伸入侧栏。内部 `advanced-mode-field` 只作为普通行项目。不要把每个字段单独绝对定位，否则显式模式、SSE 开关和 token 开关会叠在同一位置并导致文字窄列换行。

```text

├─ SCHEMA.md          # 格式规范（AI / 维护者必读）

├─ m0-config.json
├─ m1-tools.json
├─ m1-agent.json
├─ m2-routing.json
├─ m3-plan.json
├─ m3-task.json
├─ m4-background.json
├─ m4-notifications.json
```

> **API 变更后**：修改 `src/server/createHttpServer.ts` 或新增接口后，必须**重启** `npm run serve` 并 **Ctrl+F5** 刷新测试台；否则可能收到 `404` + `{"error":"未知接口"}`。`GET /api/config` 的 `capabilities` 可用来确认编排/审计等模块已加载。

## 功能页与单条用例

**功能页**顶层：`milestone`、`feature`、`featureId`、`summary`、`cases[]`。

**单条用例**必填：

| 字段 | 说明 |
| --- | --- |
| `purpose` | **测试目的**：验证什么、为何需要 |
| `title` | 短标题 |
| `method` / `path` / `input` | 请求 |
| `expect` | 期望断言（`status`、`bodyPaths` 等） |

## 复制给 AI

- **复制用例**：单条 purpose + 完整 JSON + expect 说明 + SCHEMA 引用
- **复制本页全部用例**：整页 JSON + `index.json` 登记项模板

粘贴给 AI 时可直接说：「按剪贴板格式在 `m1-tools.json` 追加一条用例」。

## 模型选择（需调用 LLM 的用例）

测试用例工作区右侧顶部有 **模型** 下拉框：

- 仅列出 `GET /api/models/check` 中 **当前可用** 的客户端
- **自动路由**：按配置策略选择模型（与主对话区「自动」一致）
- 运行内置用例、本页全部、手动验证时，对 `/api/chat`、`/api/agent`、`/api/plan`、`/api/subagent/*` 会自动注入 `clientName`（输入 JSON 已写 `clientName` 时以 JSON 为准）
- 点 **刷新模型** 可重新探测可用性

## 手动输入验证（每个功能页均有）

每个功能页底部都有 **手动输入验证** 区：

1. 在页面顶部选好 **模型**（若接口需要）
2. 下拉选择本功能相关 API，或选 **自定义 method + path**
3. 编辑 **输入** JSON、可选 **期望** JSON
4. 点击 **运行手动验证** → 下方三列展示结果（未填期望时仅展示实际输出）
5. 内置用例可点 **填入手动验证** 把该条输入/期望载入表单再改
6. **复制为正式用例** 生成 AI 可维护的 JSON 草稿

## 新增功能检查清单

1. 在对应里程碑 JSON **追加 ≥2 条**（含 `purpose`）
2. 新建功能页时：新增 `{milestone}-{feature}.json` 并在 `index.json` 按顺序登记
3. 新功能页仅在 `index.json` 登记；测试台保持单一「测试用例」入口，功能页在验收工作台左侧切换
4. 同步 `SCHEMA.md` / 本文若格式有变

当前内置功能页均保持 **不少于 2 条**网页用例；模型相关接口可在面板顶部选择 DeepSeek 等可用客户端后运行。

## 与 `npm test` 的关系

| | `npm test` | 网页功能页 |
| --- | --- | --- |
| 位置 | `tests/*.test.ts` | `public/test-cases/*.json` |
| 目的字段 | 测试名 / 注释 | 必填 `purpose` |
| 用户可见 | 终端 | 浏览器三列比对 |
